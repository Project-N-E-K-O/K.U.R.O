# 武器构筑系统设计文档 v2.0

---

## 一、系统概述

构筑系统分为两个独立层次，互相补充：

| 层次 | 名称 | 获取方式 | 效果类型 | 数量上限 |
|---|---|---|---|---|
| 构筑效果 | Build Effect | 得分三选一 / 挑战宝箱 | 数值微调 | 每种上限6个 |
| 构筑等级 | Build Level | 自动解锁（累积阈值） | 行为改变 | 固定三级 |

---

## 二、得分系统

### 得分来源

| 来源 | 得分 |
|---|---|
| 击杀普通敌人 | +10分 |
| 击杀精英敌人 | +50分 |
| 击杀Boss | +100分 |

### 触发机制

```
每积累 200分 → 触发一次三选一弹窗
全程预期触发次数 ≈ 8次（5场景标准流程）
```

---

## 三、构筑效果（Build Effect）

### 设计定位

- 数值微调型，单次效果微弱但可感知
- 可多次叠加，提供持续成长感
- 每个构筑拥有 **6个不同效果**
- 每种效果叠加上限 **4-6次**（具体由效果强度决定）

### 三选一规则

- 三张卡片可以来自同一构筑（制造决策张力）
- 三张卡片从全局效果池加权随机抽取
- 已满级构筑的效果仍可出现，标注「满级·纯数值」

### 卡片UI结构

```
┌─────────────────────────────────────┐
│  🛡 [构筑名称]           ⚡即将升级  │
│─────────────────────────────────────│
│  「效果名称」                        │  ← 第一行：本次获得的构筑效果
│  效果描述 +数值                      │
│─────────────────────────────────────│
│  构筑等级进度                        │  ← 第二行起：三级预览
│  ● Lv1「效果名」  已激活            │
│  ◐ Lv2「效果名」  3/4 ← 选中即解锁  │
│  ○ Lv3「效果名」  需6个             │
└─────────────────────────────────────┘
```

### UI交互规则

| 状态 | 图标 | 文字颜色 | 显示内容 |
|---|---|---|---|
| 已解锁 | ● 全亮 | 正常色 | 「已激活」 |
| 进行中 | ◐ 半亮 | 正常色 | 「X/阈值」进度 |
| 即将升级 | ◐ 高亮 | 强调色 | 「选中即解锁」 |
| 未解锁 | ○ 暗色 | 灰色 | 「需X个」 |

- 鼠标悬停实时预览选择后的状态变化
- 选择后触发升级播放点亮动画

## 四、构筑等级（Build Level）

### 设计定位

- 行为改变型，质变感强，不随叠加次数放大
- 每个构筑固定三级，自动解锁
- 不占用三选一名额，达到阈值后后台自动触发

### 解锁阈值

| 等级 | 所需同类构筑效果数量 |
|---|---|
| Lv1 | 1个 |
| Lv2 | 4个 |
| Lv3 | 6个 |

> 加入挑战宝箱后单一楼层全程可额外获得1-2个构筑效果，
> 阈值1/4/6确保宝箱的额外资源恰好填补正常节奏的空档，
> 专注单构筑的玩家仍需到第4-5场景才能满级。

### 效果设计原则

```
✓ 时间控制型  击晕时间延长 / 减速时间延长
✓ 计数触发型  触发门槛降低 / 触发后效果扩大
✓ 资源型      充能次数增加 / 冷却时间缩短
✗ 纯数值型    避免+X%伤害、+X%攻速等无上限叠加
```

## 六、挑战宝箱系统

### 设计定位

- 每个关卡有 **50-60%概率** 出现挑战
- 完成挑战必定开箱，失败无惩罚
- 宝箱给予构筑效果、武器、金币
- 宝箱给出的构筑效果不计入得分触发计数（两套系统独立）

### 挑战类型原则

```
✓ 顺手型（不改变正常打法）
  - 本关卡内击杀X只敌人
  - 不受伤通过本关卡
  - 在X秒内清空本关卡

✗ 特意型（破坏割草节奏）
  - 找到隐藏开关
  - 用特定武器击杀敌人
  - 保护某个目标
```

### 数值影响评估

```
原本资源量：三选一约8次
宝箱额外资源：约2-3个构筑效果
总计：约10-11个构筑效果获取机会

对应阈值1/4/6：
  专注单构筑 → 6次选同类 → 第4-5场景满级
  副线构筑   → 剩余4-5次 → 达到2级
```

## 七、数值膨胀控制

### 构筑效果侧

- 每种效果叠加上限4-6次（由效果强度决定）
- 数值提升控制在基础值5-10%以内
- 满级后同类效果仍可获得但标注「满级·纯数值」

### 敌人成长侧

| 场景 | 血量倍率 | 伤害倍率 |
|---|---|---|
| 场景1 | ×1.0 | ×1.0 |
| 场景2 | ×1.3 | ×1.2 |
| 场景3 | ×1.6 | ×1.4 |
| 场景4 | ×2.0 | ×1.7 |
| 场景5 | ×2.5 | ×2.0 |

- 玩家血量上限保持12点不变
- 敌人成长与玩家构筑成长大致同步，保持战斗节奏稳定


## 九、数值体系（×10扩大基数）

### 为什么必须×10

当前数值体系下构筑效果的数值提升完全失效：

```
当前体系
  玩家基础攻击    1
  武器伤害        2-10
  实际伤害        3-11点

  构筑效果5%提升  = 3-11 × 0.05 = 0.15-0.55点
  四舍五入后      = 0点（完全无感）
  最好情况        = 1点（勉强可见）

结论：个位数基数下，百分比提升在整数运算中归零
```

### ×10后的数值体系

| 数值项目 | 原始值 | ×10后 |
|---|---|---|
| 玩家基础攻击 B | 1 | 10 |
| 玩家基础攻击上限 | 10 | 100 |
| 武器伤害 W | 2-10 | 20-100 |
| 玩家血量上限 | 12 | 120 |
| 普通敌人血量 | 15-30 | 150-300 |
| 精英敌人血量 | 60-80 | 600-800 |
| 普通道具伤害 | 2/6/10 | 20/60/100 |
| 敌人攻击力 | 2-10 | 20-100 |

### ×10后构筑效果数值提升验证

```
实际伤害范围（×10体系）
  最低配  B=10 + W=20×N=1 = 30点
  标准配  B=10 + W=50×N=2 = 110点
  高配    B=50 + W=80×N=3 = 290点

固定值提升方案
  基础伤害+2点   → 占标准配110点的1.8%，感知合理
  基础伤害+5点   → 占标准配110点的4.5%，感知明显
  攻击速度+0.05  → 固定值，全程稳定
```

### 为什么不用百分比而用固定值

```
百分比提升的问题
  早期实际伤害30点  × 5% = 1.5点  → 几乎无感
  后期实际伤害600点 × 5% = 30点   → 过于强力
  同一构筑效果，早期和后期的绝对增益差20倍
  → 数值体验断层严重

固定值提升的优势
  基础伤害+5点
  早期实际伤害30点  +5 = 增加16.7%  → 早期感知明显
  后期实际伤害290点 +5 = 增加1.7%   → 后期微调稳定
  随着玩家变强，单次效果的相对权重自然降低
  → 天然的数值收敛，不需要额外控制
```

### 构筑效果固定值参考范围

| 效果类型 | 推荐固定值 | 叠加上限 | 总计最大增益 |
|---|---|---|---|
| 基础伤害 | +2点 | 6次 | +12点 |
| 武器伤害 | +5点 | 5次 | +25点 |
| 攻击速度 | +0.05 | 4次 | +0.20 |
| 回血量 | +1点 | 4次 | +4点 |
| 冷却时间 | -0.3秒 | 4次 | -1.2秒 |
| 充能次数 | +1次 | 2次 | +2次 |
| 控制时长 | +0.3秒 | 4次 | +1.2秒 |

### 伤害公式（最终版）

```
实际伤害 = round((B + W×N) × MT)

B   玩家基础攻击（10-100，通过超能力成长）
W   武器伤害（20-100）
N   攻击段数（1-5）
MT  武器类型倍率
	拳击类 × 1.3
	挥砍类 × 1.1
	戳刺类 × 1.0
	投掷类 × 0.8

武器伤害W与段数N的约束
  W=80-100  N ≤ 2
  W=50-70   N ≤ 3
  W=20-40   N ≤ 4
  W=10-20   N ≤ 5
```
